Step 1: Install OpenClaw (10 Minutes)

You need a computer that stays on (or a cheap server). Mac, Linux, or Windows with WSL2.

What you need first:
Node.js version 22
or higher
A terminal (Terminal
on Mac, any terminal on Linux)
An API key
from Anthropic (claude.ai) or OpenAI
Install it:
1
Open your terminal
2
Run: `npm install -g openclaw@latest`
3
Run: `openclaw onboard --install-daemon`
4
Follow the wizard. It asks which AI provider to use. Pick Anthropic (Claude) for best results.
5
Paste your API key when asked
Check it's running:

Run: openclaw gateway status

Key Takeaway: One command to install, one wizard to configure. The daemon flag makes it run forever in the background.

Step 2: Configure the Heartbeat Schedule

Heartbeat is controlled by one config file: ~/.openclaw/openclaw.json

Open it in any text editor and add the heartbeat settings.

Basic config (checks every 30 minutes, sends alerts to your last active chat):
{
  "agents": {
    "defaults": {
      "heartbeat": {
        "every": "30m",
        "target": "last",
        "model": "anthropic/claude-sonnet-4-5"
      }
    }
  }
}
With business hours (only runs 9am-10pm, your timezone):
{
  "agents": {
    "defaults": {
      "heartbeat": {
        "every": "30m",
        "target": "last",
        "model": "anthropic/claude-sonnet-4-5",
        "activeHours": {
          "start": "09:00",
          "end": "22:00",
          "timezone": "Australia/Sydney"
        }
      }
    }
  }
}
Send alerts to a specific channel (e.g., Telegram):
{
  "agents": {
    "defaults": {
      "heartbeat": {
        "every": "1h",
        "target": "telegram",
        "model": "anthropic/claude-sonnet-4-5"
      }
    }
  }
}

Important: Use claude-sonnet-4-5 for heartbeat, not Opus. Sonnet is cheaper and fast enough for monitoring tasks. Opus would cost 10-20x more for no real benefit.

🔑

Key Takeaway: Set the interval based on how urgently you need to know. Every 30 minutes for ads and leads. Every hour or two for project tracking.

Step 4: Connect Your Channels

Heartbeat alerts need somewhere to go. Connect WhatsApp, Telegram, or Slack so you get pinged on the device you actually check.

WhatsApp (most common):
1
Run: `openclaw channels login`
2
A QR code appears in your terminal
3
Open WhatsApp on your phone, go to Settings > Linked Devices > Link a Device
4
Scan the QR code
5
Done. Message the agent to test.
Telegram:
  1. Create a bot on Telegram (message @BotFather, follow the prompts)
  2. Copy the bot token
  3. Run: openclaw channels add telegram --token YOUR_BOT_TOKEN
  4. Approve the pairing when prompted
Slack:
1
Create a Slack app at api.slack.com
2
Add a bot token with the right permissions
3
Run: `openclaw channels add slack --token xoxb-YOUR-SLACK-BOT-TOKEN`

Step 5: Pair It With n8n for Best Results

Here's what most guides won't tell you. OpenClaw's heartbeat can't directly pull data from Meta Ads, HubSpot, or your project management tool without extra setup.

How the combo works:
1
n8n runs a scheduled workflow** that pulls data from your ad account, CRM, or project tool every hour
2
n8n saves the data** to a Google Sheet or a JSON endpoint
3
OpenClaw's heartbeat reads that data** and evaluates whether anything needs your attention
4
If something's off, it alerts you.** If not, silence.
Example n8n workflow:
1
Schedule trigger: every hour
2
HTTP Request: pull today's ad spend from Meta Marketing API
3
Google Sheets: write the numbers to a "Live Metrics" tab
4
OpenClaw's heartbeat reads the Sheet and evaluates

Watch Out

Common Mistakes

Mistake 1: Using Opus for heartbeat+

Claude Opus costs 10-20x more than Sonnet per run. At 48 runs per day, that's $5-30/day vs $0.24/day. Use Sonnet for heartbeat. Save Opus for complex reasoning tasks.

Mistake 2: Making HEARTBEAT.md too long+

Every word gets injected into every heartbeat prompt. A 1,000-word checklist running 48 times a day burns tokens fast. Keep it under 500 words. Be specific about what matters. Cut everything else.

Mistake 3: Not setting active hours+

Without activeHours, heartbeat runs 24/7. That means you get alerts at 3am that could have waited until morning. Set business hours in the config. Your sleep is worth more than instant alerts.

Mistake 4: Giving it write access before testing read access+

Start by having heartbeat only read and report. Don't let it send emails, update records, or take action until you've watched it for at least a week. Build trust gradually.

Mistake 5: Editing config while the gateway is running+

The gateway keeps config in memory. If you edit files while it's running, your changes get overwritten. Always: stop the gateway, make edits, restart the gateway.
